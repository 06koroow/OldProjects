Lavanda

Before creating a ticket for a new issue, it’s always a good idea to search for an existing ticket to see if the issue has already been reported. If not, to create a new ticket:

  • In the main menu, click Operations, then Tickets
  • On the Ticket page, on the top right-hand corner, click New Ticket
  • In the ticket creation page, select a property code and priority and choose a due date by clicking on the calendar
  • Add a title and description to your ticket that fully describes the issue you’re reporting and what needs to happen next
  • Assign the ticket to a relevant team queue and category
  • Select an owner for the ticket. This should be the user that’s responsible for progressing the issue towards resolution
  • Hit Create This Ticket to save your ticket and notify the owner.

    Information to include on Tickets

  1. 1. Property: The Veeve Property ID as assigned on PMS
  2. 2. Status: Set to New as standard, changed to In Progress when the job is booked.
  3. 3. Priority: How urgently does the issue need to be fixed?
  4. 4. Due Date: the prospective due date for the ticketed issue to be resolved, when a task is created. Based on the priority assigned above. This is an estimate based on priority until the relevant department reviewed the resolution steps and can assign a proper due date.
  5. 5. Title: A short summary of the issue reported. E.g. “WIFI Cutting out at regular intervals.”
  6. 6. Job Description: A full, fleshed out description of the issue.
  7. 7. Category: Assigning the issue type / segment to a ticket
  8. 8. Tags: Key descriptive word to associate ticket or tasks to an activity (Tags can only be added once a ticket is saved and created) e.g. if the task includes fridge the tag would be “Appliances “
  9. 9. Team Queue: This will usually be the team carrying the ticket to completion, whilst also sending notifications of ticket updates to members of that team.
  10. 10. Notify All Members: Checking this button will notify all the members of a particular team that an issue has been added to their queue
  11. 11. Owner: The person responsible for ensuring the ticket is completed (including task tracking)
  12. 12. Create This Ticket: Once you have input all the information, click this button to create the ticket and add it to the selected team’s Job Queue

Once a ticket has been created in Lavanda, it can be found from the Tickets page:

  • In the main menu, click Operations, then Tickets
  • Use the search bar at the top of the Tickets page to locate the ticket you’re looking for. You can click into the search bar to apply filters to narrow down your search, or to see a wider range of tickets.
  • To save a certain filter combination, click the More Actions button to the right of the search bar and hit Save Current Filters.

    Information to Include on Tasks

  1. 1. Type of Task: Which type of task is being booked?
  2. 2. Task Title: A short description of what needs to be completed.
  3. 3. Job Description: A full description of what needs to be completed, including access details, and any parts required.
  4. 4. Tags: These are keywords assigned to the tasks,
  5. 5. Team Queue: the department that will be resolving this request.
  6. 6. Assigned To: This is the individual that is seeing the resolution steps through to completion.
  7. 7. Create This Task: This button will create the task, add it to the relevant team queue and assign it to the team member defined above.

Once a ticket has been created in Lavanda, it can be found from the Tickets page:

  • In the main menu, click Operations, then Tickets
  • Use the search bar at the top of the Tickets page to locate the ticket you’re looking for. You can click into the search bar to apply filters to narrow down your search, or to see a wider range of tickets.
  • To save a certain filter combination, click the More Actions button to the right of the search bar and hit Save Current Filters.

    Information to Include on Tasks

  1. 1. Segment Search: These are combinations of filters, pre-defined for ease of locating tasks
  2. 2. Currently Selected Filters: These are the filters currently selected and a search bar to add more filters.
  3. 3. Ticket Title: The title of the ticket as assigned when the ticket was created.
  4. 4. Team Queue and Assigned To: Lists which team and individual the ticket is assigned to.
  5. 5. New Ticket Button: This button will create a new ticket.
  6. 6. Sort By: This will change how the list is sorted and what order the tickets appear in.
  7. 7. Status, Watchers and Last Updated: This will show the status of the tickets, whether anyone is watching the ticket, and when the ticket was last updated.
